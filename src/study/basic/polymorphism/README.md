## 📦 Java 다형성(Polymorphism) 실습
추상 클래스를 상속받은 다양한 하위 클래스들을 다형성을 활용해 통합 관리하는 방법을 실습

### 2.핵심 설계 구조
- **추상 클래스 (Delivery)**: 공통 필드(item, address)와 공통 로직(printInfo)을 제공하며, 배송 방식별로 달라지는 요금과 기간은 추상 메서드로 선언되었기에 자식에서 반드시 구현해야 함
- **다형성**: Delivery[] 배열을 생성하여 Standard, Express, Overnight 객체를 한꺼번에 담고, 같은 메서드(calculateFee, estimateDays)를 했지만, 서로 다른 결과가 출력 됨.

### 3. 배송 방식별 구현 상세
| 배송 유형 | 요금      | 예상 소요일 | 특징       |
|-------|---------|--------|----------|
| 일반 배송 | 3,000원  |3일| 기본 배송 모델 |
| 빠른 배송 | 5,000원  |1일| 익일 도착 목표 |
| 새벽 배송 | 10,000원 |0일|당일 배송 안내 문구 추가|

### 4. 주요 학습 포인트
- **추상 메서드 구현**: 자식 클래스들이 각자의 비즈니스 로직에 맞게 calculateFee()와 estimateDays()를 재정의(오버라이딩) 
- **통합 타입 관리**: 서로 다른 성격의 배송 객체들을 Delivery라는 하나의 타입으로 묶어서 관리하므로, 다른 배송 타입이 추가되어도 수정에 불편함이 없음
