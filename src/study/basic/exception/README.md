## ⚠️ Java 예외 처리(Exception) 실습
사용자 정의 예외 클래스를 생성하고, 예외 발생 시 호출 스택(Call Stack)을 추적하여 디버깅하는 방법을 실습

### 1. 주요 설계 및 기능
- **사용자 정의 예외 (InvalidScoreException)**: Exception 클래스를 상속받아, 성적 관리 시스템에 특화된 예외 상황(범위 초과 등)을 처리
- **예외 던지기 (throws)**: setScore 메서드에서 잘못된 값이 들어올 경우 직접 예외를 발생시켜 호출한 곳으로 처리를 위임
- **예외 처리 (try-catch)**: main 메서드에서 예외를 잡아 프로그램이 비정상 종료되지 않도록 관리하고 오류 메시지를 출력

### 2. 주요 학습 포인트
- **호출 스택 추적 (`printStackTrace()`)**: 예외 발생 시 main → setScore와 같이 메서드 호출 경로를 역순으로 보여주어, 복잡한 코드 내에서도 에러의 원인지를 빠르게 파악할 수 있다. (서버 입장)
- **예외 메시지 관리 (`getMessage()`)**: 예외 생성 시 전달한 상세 사유를 사용자에게 보여줌으로써 어떤 데이터가 문제를 일으켰는지 알려줌 (클라이언트 입장)
